# 变更提案：优化分镜头向导界面布局

## 变更 ID
`optimize-wizard-ui-layout`

## 概述

将"导入分镜头向导"和"导出分镜头向导"的用户界面从当前的竖版显示模式统一调整为横版显示模式，确保所有现有功能保持不变且正常运行。

## 动机

### 当前问题
- Steps 组件采用竖版布局，在对话框中占用过多垂直空间
- 步骤指示器和描述文字竖向排列，导致对话框高度过大，需要频繁滚动
- 横向空间利用率不足，视觉层次不够清晰

### 期望结果
- 采用横版布局，步骤指示器横向排列，更符合向导式交互的通用设计模式
- 减少垂直空间占用，提升视觉舒适度
- 保持与整体产品设计规范的一致性

## 影响范围

### 涉及组件
- `src/components/ui/steps.tsx` - Steps 和 Step 组件
- `src/features/storyboard/components/template-import-dialog.tsx` - 导入对话框
- `src/features/storyboard/components/template-export-dialog.tsx` - 导出对话框

### 受影响的用户流程
- 导入分镜头向导（6步骤）
- 导出分镜头向导（5步骤）

## 设计考虑

### 布局方向
- **当前**: 竖版布局（步骤垂直排列）
- **目标**: 横版布局（步骤水平排列）

### 视觉元素
- 步骤指示器（圆圈）保持现有设计
- 连接线从竖向改为横向
- 步骤标题和描述调整位置

### 响应式考虑
- 在小屏幕上自动切换为紧凑模式
- 确保在不同断点下都有良好的显示效果

## 依赖关系

- 无外部依赖
- 依赖现有的 Steps 组件和对话框组件

## 时间估算

- **Steps 组件改造**: 1-2 小时
- **对话框集成调整**: 1 小时
- **测试和验证**: 1 小时
- **总计**: 3-4 小时

## 验收标准

- [ ] Steps 组件支持横版布局
- [ ] 导入/导出对话框正确使用横版布局
- [ ] 所有现有功能保持正常运行
- [ ] 响应式设计在各断点正常工作
- [ ] 视觉效果符合产品设计规范
