# Proposal: Compact Storyboard List Layout

## Summary

优化"分镜头清单"页面的表格布局，采用紧凑布局模式，减少字段间距，同时对长文本字段进行截断处理，提高信息密度和浏览效率。

## Motivation

当前分镜头清单表格存在以下问题：
1. 字段间距较大，导致在有限屏幕空间内显示的数据行数较少
2. "画面描述"和"音效说明"等长文本字段占用过多空间
3. "项目"列默认显示，但在单项目视图下意义不大

## Proposed Changes

### 1. 紧凑布局
- 减少 Table 单元格的 padding 和间距
- 使用更紧凑的行高设置

### 2. 画面描述字段
- 显示简化为最多 8 个字符
- 超出部分使用省略号 `...` 替代
- 保留悬停显示完整内容的功能

### 3. 项目字段
- 默认隐藏"项目"列（`projectId` column visibility 默认设为 `false`）
- 用户可通过"显示列"下拉菜单手动开启

### 4. 音效说明字段
- 显示简化为最多 5 个字符
- 超出部分使用省略号 `...` 替代
- 保留悬停显示完整内容的功能

## Scope

| Area | Change Type |
|------|-------------|
| `shot-list-table.tsx` | Modified |

## Acceptance Criteria

1. [ ] 表格采用紧凑布局，行间距明显减少
2. [ ] "画面描述"列最多显示 8 个字符，超出用省略号
3. [ ] "项目"列默认隐藏
4. [ ] "音效说明"列最多显示 5 个字符，超出用省略号
5. [ ] 悬停仍可查看完整内容
6. [ ] 构建无错误
