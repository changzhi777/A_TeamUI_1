# Proposal: Enhance Character Design with Six-Dimensional Template

## Overview
增强角色设计模块，基于完整六维模板整合，实现角色数据的文件夹式组织结构，并改进资产库同步和UI功能。

## Motivation
当前角色设计功能存在以下问题：
1. 角色卡片缺少项目名称（剧本名称）显示
2. 角色素材没有按文件夹结构组织导出
3. 资产库同步缺少"上传者"信息显示
4. 多视角图片卡片缺少手动上传功能
5. 缺少"自定义卡片"功能用于扩展信息
6. 角色属性信息标签仍为英文
7. 生成提示词功能缺少"提示优化"能力

## Proposed Changes

### 1. 角色卡片增强
- 在角色卡片上显示所属项目名称（剧本名称）
- 优化卡片信息布局

### 2. 文件夹式数据组织
- 按角色编号创建文件夹
- 多视角图片保存到 `views/` 子目录
- 服装变体图片保存到 `costumes/` 子目录
- 语音文件保存到 `voice/` 子目录
- 属性信息和提示词保存为 JSON 文件

### 3. 资产库同步增强
- 显示当前用户作为"上传者"
- 在资产卡片上显示上传者信息

### 4. 多视角图片上传
- 为每个视角卡片添加本地上传功能
- 支持直接上传本地图片作为角色视角

### 5. 自定义卡片功能
- 支持添加自定义信息字段
- 支持自定义字段的数据保存和展示

### 6. 属性信息中文化
- 将角色属性标签翻译为中文
- 年龄、性别、身高、职业、发色、瞳色、体型等

### 7. 提示词优化功能
- 基于六维角色模板整合提示词
- 提供提示词优化建议
- 一键生成完整角色描述

## Impact
- **character-design** spec: 新增多项功能需求
- **asset-management** spec: 新增上传者显示需求

## Dependencies
- 现有角色类型系统
- 现有资产库同步服务
- 现有文件导出功能

## Timeline
- Phase 1: 角色卡片增强和属性中文化
- Phase 2: 文件夹导出和资产库上传者
- Phase 3: 多视角上传和自定义卡片
- Phase 4: 提示词优化功能
